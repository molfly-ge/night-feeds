31 твит за сутки. Практик AI-автоматизации делится опытом: от ручной координации агентов до полностью автономных воркфлоу через n8n.

Тред: от ручных агентов к автономным воркфлоу — шестичастный тред о пути автоматизации. Начинал с TypeScript-воркфлоу, которые требовали постоянного ручного надзора. Прорыв случился, когда агенты стали передавать контекст напрямую через n8n-вебхуки вместо копипаста между сессиями. Сейчас контент-пайплайн полностью автономен: research-агент кормит writing-агента, тот триггерит постинг. Агенты даже дебажат друг друга — writing-агент ловит ошибки форматирования research-агента. Claude Code как ежедневный инструмент — использует Claude Code как основной инструмент разработки. Построил целую систему Telegram-ботов, просто описывая желаемое и итерируя. Отмечает сдвиг: теперь больше времени уходит на продумывание задачи и ревью, чем на написание кода. Сравнивает: "как быть архитектором вместо плотника". Твит, Твит

Локальные модели и экономика — запускает локальные модели на домашнем сервере с 4090. Причина: rate limits облачных API не подходят для агентов 24/7. Интересуется Gemma 4 и DGX Spark. Вернулся к локальным моделям после повышения тарифов. Твит, Твит, Твит

Goose vs Claude Code — попробовал Goose: локальный запуск работает, но навигация между файлами неудобна по сравнению с Claude Code. Впрочем, "$180/месяц — это $180/месяц". Спрашивает, работает ли Goose с локальными Llama-моделями. Твит, Твит

Компания одного человека — запускает 6 AI-агентов на домашнем сервере, которые заменяют целые команды. Считает, что большинство фаундеров всё ещё по привычке думают, что нужно нанимать людей первым делом. Твит

Obsidian + Claude для баз знаний — использует Claude для организации сырых данных в базы знаний. Считает это ценнее, чем генерация кода. Твит, Твит

Специализация агентов — разделяет агентов по узким функциям: Telegram-бот ничего не знает о контент-стратегии, контент-агент не знает про n8n. Узкий фокус даёт лучшие результаты. Твит

Cursor для итераций, Claude для архитектуры — перешёл на Cursor для повторяющегося редактирования файлов. Claude Code быстро сжигает токены при итерациях над одним файлом: контекстное окно забивается дублирующими диффами. Твит

Токены и масштабирование — ключевая мысль: большинство людей при первой неудаче думают "модель недостаточно умна", хотя на самом деле проблема — "недостаточно токенов". Больше think-time = лучше результат. Твит
